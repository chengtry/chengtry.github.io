ES6中引入了let和const关键字
少用var多用let

1.在块级作用域里有效（比如循环遍历内部就可以用let定义类型变量）
<script type="text/javascript">
        {
            let a = 0;
            var b = 1;
            console.log("a" + a);
            console.log("b" + b);
        }
        //console.log(a); // 报错：a is not defined
        console.log(b);
</script>


2.不能重复声明
<script type="text/javascript">
        {
            let a = 0;
            var b = 1;
            let a = 2;
            var b = 2;
            console.log("a" + a);  // 报错：Identifier 'a' has already been declared
            console.log("b" + b);
        }
        //console.log(a);
        console.log(b);
    </script>

3.不存在变量提升
<script type="text/javascript">
        console.log(a);  // 不报错，输出undefined（var使得变量提升）
        console.log(b); // 报错：Cannot access 'b' before initialization
                var a = 0;
        let b = 1;
    </script>

MVVM（Model-View-ViewModel）是一种软件架构设计模式，由微软 WPF（用于替代 WinForm，以
前就是用这个技术开发桌面应用程序的）和 Silverlight（类似于 Java Applet，简单点说就是在浏览器上
运行的 WPF） 的架构师 Ken Cooper 和 Ted Peters 开发，是一种简化用户界面的事件驱动编程方式。
由 John Gossman（同样也是 WPF 和 Silverlight 的架构师）于 2005 年在他的博客上发表。


1.M:Model：模型层，这里表示JavaScript对象，对应data中的数据
2.V:View：视图层，这里表示DOM（HTML操作的元素），模板
3.VM:ViewModel：连接视图和数据的中间件，Vue.js就是MVVM中ViewModel层的实现者，Vue实例对象

vm身上所有的属性及Vue原型上所有的属性，在Vue模板中都可以直接使用

ViewModel

  • ViewModel能够观察到数据的变化，并对视图对应的内容进行更新
  • ViewModel能够监听到视图的变化，并能够通知数据发生变化
  • Vue.js是一个MVVM的实现者，它的核心就是实现了DOM监听与数据绑定
